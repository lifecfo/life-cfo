# Life CFO — User Interaction Modes

Last updated: 2026-03-11

Life CFO is designed to support different **depths of financial reasoning**, not rigid user categories.

The same person may interact with the system in different ways at different times.

For example:
- someone may want a quick reassurance most days
- but occasionally need deep analysis for a major decision

The system should adapt by revealing **more or less depth**, while keeping the interface calm and legible.

---

# 1. Low-Friction Checker

This user wants a quick answer to a simple question:

"Are we okay?"

They do not want to:
- learn the system
- manage data
- run analysis
- explore dashboards

They want reassurance that nothing important is being missed.

### Product needs

- one clear orientation surface
- minimal interaction required
- simple explanations
- optional depth, never required depth

### Success state

The user opens Life CFO briefly, understands the current state of things, and closes the app with confidence.

They should feel:

"I don’t really use it much, but it helps."

---

# 2. Overwhelmed Avoider

This user is carrying significant mental load and cannot take on additional responsibility.

They may avoid financial tools because they fear:
- finding problems
- being behind
- needing to take action

Life CFO must remove pressure the moment it appears.

### Product needs

- no backlog pressure
- no progress indicators
- no "you have X items" signals
- calm language
- reassurance that nothing urgent is being missed
- permission to disengage

### Success state

Opening the app makes the user feel **less overwhelmed**, even if they do very little.

---

# 3. Question-Driven Decision Maker

This user comes to Life CFO with a specific financial question.

Examples include:

- "Can we afford this?"
- "Should we wait?"
- "Why does money feel tight right now?"
- "What would change if we did this?"

### Product needs

- grounded answers based on real financial data
- clear explanations
- visible assumptions
- trade-off analysis
- scenario exploration when relevant

### Success state

The user receives a clear answer and understands the reasoning behind it.

They should not feel the need to repeatedly re-think the same question.

---

# 4. High-Agency / Power User

Some users will want deeper financial reasoning and analytical capability.

Examples may include:

- professionals
- founders
- investors
- financially sophisticated households
- analytically minded users
- high-net-worth individuals

These users may want to:

- explore multiple scenarios
- compare financial outcomes
- examine assumptions
- understand deeper structural patterns in their finances
- generate structured outputs or summaries

### Product needs

Life CFO should support deeper analysis without increasing interface complexity.

Capabilities may include:

- scenario modelling
- structured comparison outputs
- deeper financial breakdowns
- exportable reasoning summaries
- the ability to interrogate conclusions

However, these capabilities must remain **invited depth**, not the default surface.

### Success state

The system feels capable and intelligent without becoming overwhelming.

Power users should feel that Life CFO can reason at their level while still maintaining clarity.

---

# Economic Diversity

Life CFO must work equally well for households with:

- very limited financial resources
- moderate financial complexity
- high net worth
- complex income structures
- multiple assets or investments

The value of the system is **clarity of decisions**, not the size of the balance sheet.

A single parent managing tight cash flow and an investor managing multiple properties both benefit from clearer financial reasoning.

Life CFO should respect both contexts.

---

# Design Principle — Progressive Depth

Life CFO should scale in **depth of reasoning**, not in **interface complexity**.

This means:

- casual users see calm summaries
- question-driven users see analysis
- power users can explore deeper reasoning and scenarios

The same system supports all of these interactions without requiring different modes or separate products.

Depth appears only when the user invites it.

---

# Unifying Product Goal

Life CFO should help users feel:

- informed
- calmer
- less alone with financial decisions
- confident that nothing important is being missed

The product succeeds when it reduces cognitive load rather than increasing it.

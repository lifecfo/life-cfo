# Engineering Rules

These rules protect the Life CFO architecture.

1. Household is the global access boundary.

2. All financial data must be household-scoped.

3. API routes must enforce authorization.

4. Client-side Supabase queries should not bypass household boundaries.

5. AI outputs must remain analytical, not directive.

6. The system must not perform autonomous financial actions.

7. UI language should remain calm, clear, and non-alarmist.

These rules override convenience.

Life CFO — Canonical Answer Template (LOCKED)
Purpose
Deliver a clear, trustworthy decision outcome that:
•	Stops mental looping
•	Reduces cognitive load
•	Allows confident action or confident deferral
•	Is explainable without being overwhelming
The primary success signal is:
The user feels lighter after reading the first sentence.
________________________________________
STRUCTURE (IN ORDER — NEVER REORDER)
________________________________________
1️⃣ Verdict (Required, Always First)
One sentence. Bold. Plain language.
This is the answer.
Everything else supports it.
Rules
•	Must directly address the question
•	Must be understandable in isolation
•	Must not include numbers unless absolutely essential
•	Must not hedge
•	Must not include “from what I can see”, “it depends”, or “likely”
Allowed verdict types
•	Yes
•	No
•	Yes, but needs planning
•	Not yet
•	This needs attention
•	This is fine for now
Examples
•	“Yes — this fits comfortably within your current position.”
•	“This would put pressure on your finances this year.”
•	“This is achievable, but only with planning.”
•	“Not yet — this would be risky without changes.”
👉 If this sentence is removed, the answer has failed.
________________________________________
2️⃣ Contextual Clarifier (Optional, Max 1 sentence)
Only include if it genuinely adds clarity.
Purpose
•	Explain why the verdict is what it is
•	Provide reassurance without reopening the decision
Rules
•	One sentence max
•	No new concepts
•	No extra numbers
•	No advice
Examples
•	“This is because it would rely on absorbing a large one-off cost without a buffer.”
•	“This assumes no major changes to income or expenses.”
________________________________________
3️⃣ Supporting Facts (Optional, Clearly Labeled)
Facts exist to justify, not to lead.
Heading (fixed copy):
What this is based on:
Rules
•	Bullet points only
•	Concrete, observable facts
•	No interpretation
•	No projections
•	Max 3–4 bullets
Examples
•	Available balances: A$46,700
•	Monthly commitments: A$7,705
•	Upcoming bills in next 30 days: 6
If facts are incomplete, this section may be omitted entirely.
________________________________________
4️⃣ Status Framing (Implicit, Not Primary)
Status should be conveyed by language, not UI chrome.
If a label exists, it must:
•	Mirror the verdict
•	Never introduce new information
•	Never appear before the verdict
Preferred:
•	Status is implicit in the verdict sentence
Acceptable (secondary):
•	“Needs planning”
•	“All clear for now”
•	“Requires attention”
❌ Avoid binary or alarming labels unless truly warranted.
________________________________________
5️⃣ Optional Depth (Collapsed by Default)
These sections exist to make strong answers safe.
5a. What would change this?
Purpose: Protect against false certainty
Rules:
•	3 bullets max
•	Concrete triggers only
•	No hypotheticals
Example:
•	If a new bill is added
•	If income timing changes
•	If a large one-off expense appears
________________________________________
5b. Assumptions
Purpose: Transparency, not liability
Rules:
•	Plain language
•	No disclaimers
•	No legal tone
Example:
•	Bills and due dates are up to date
•	Account balances are current
•	No large untracked expenses are pending
________________________________________
6️⃣ Action Options (Never Required)
Actions must feel like relief, not obligation.
Rules
•	Never auto-commit
•	Never imply urgency
•	Never default to saving
Allowed actions
•	Ask a follow-up
•	Save as note
•	Save as decision
•	Leave it
Canonical copy (locked):
“Want me to hold onto this?”
________________________________________
7️⃣ Follow-ups (Context-Preserving)
Follow-ups must:
•	Keep the original answer visible
•	Reuse prior context implicitly
•	Never restart the mental loop
The follow-up answer must re-enter at Step 1 (Verdict).
________________________________________
🚫 Explicitly Disallowed
•	Facts before verdict
•	Multiple verdict sentences
•	Hedging language in the verdict
•	“It depends” as an answer
•	Reassurance duplicated in multiple places
•	Pills or badges that look clickable or ambiguous
•	Asking the user to decide instead of deciding
________________________________________
🎯 North Star Test (Final Gate)
Before shipping any answer, ask:
1.	Can the user stop thinking after the first sentence?
2.	Would a stressed parent understand this in 3 seconds?
3.	Is uncertainty handled after relief, not before?
4.	Does this reduce vigilance instead of increasing it?
If any answer is “no” → it violates the Life CFO standard.
